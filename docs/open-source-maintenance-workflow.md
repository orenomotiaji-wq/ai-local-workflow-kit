# Open Source Maintenance Workflow

This document describes a simple AI-assisted maintenance workflow.

## 1. Triage

Review issues and classify them as:

- bug
- feature request
- documentation
- question
- security concern

## 2. Prepare a report

Before modifying files, generate a report that includes:

- target files
- likely cause
- proposed fix
- risk level
- test plan

## 3. Backup

Create backups before modifying files.

## 4. Minimal change

Apply the smallest safe change.

## 5. Verify

Run available tests or manual checks.

## 6. Log

Save what changed and why.
